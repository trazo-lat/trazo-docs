Trazo uses a **schema-per-tenant** PostgreSQL isolation model:

- The **`public` schema** holds platform-wide tables that are shared across all tenants: users, tenants, roles, memberships, sessions, and service principals.
- Each **tenant schema** (e.g., `tenant_acme`, `tenant_globex`) holds tables that are fully isolated per tenant: events, audit logs, notifications, and any tables created by domain modules.

All primary keys are `UUID` generated by `gen_random_uuid()`. All timestamps are `TIMESTAMPTZ` stored in UTC.

---

## Public Schema

### `users`

Platform-level user accounts. Users are created once and can hold memberships in multiple tenants.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK, `DEFAULT gen_random_uuid()` | Unique user identifier |
| `email` | `VARCHAR(320)` | UNIQUE, NOT NULL | Email address; used for login and contact |
| `phone` | `VARCHAR(32)` | | E.164 phone number |
| `name` | `VARCHAR(255)` | NOT NULL | Display name |
| `avatar_url` | `TEXT` | | URL to the user's profile image |
| `locale` | `VARCHAR(16)` | | BCP 47 locale code (e.g., `en-US`) |
| `timezone` | `VARCHAR(64)` | | IANA timezone (e.g., `America/New_York`) |
| `password_hash` | `TEXT` | | bcrypt hash; `NULL` for OAuth-only accounts |
| `status` | `VARCHAR(32)` | NOT NULL, DEFAULT `'active'` | `active`, `invited`, or `suspended` |
| `email_verified` | `BOOL` | NOT NULL, DEFAULT `false` | Whether the email has been confirmed |
| `metadata` | `JSONB` | DEFAULT `'{}'` | Arbitrary user-level metadata |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Creation timestamp |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Last modification timestamp |

### `tenants`

One row per customer organization. Controls the schema namespace and branding.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique tenant identifier |
| `name` | `VARCHAR(255)` | NOT NULL | Display name of the organization |
| `subdomain` | `VARCHAR(63)` | UNIQUE, NOT NULL | URL-safe subdomain (e.g., `acme`) |
| `schema_name` | `VARCHAR(63)` | UNIQUE, NOT NULL | PostgreSQL schema name (e.g., `tenant_acme`) |
| `logo_url` | `TEXT` | | URL to the tenant's logo |
| `theme` | `JSONB` | DEFAULT `'{}'` | Custom branding tokens (colors, fonts) |
| `config` | `JSONB` | DEFAULT `'{}'` | Tenant-level feature configuration |
| `status` | `VARCHAR(32)` | NOT NULL, DEFAULT `'active'` | `active`, `suspended`, or `deleted` |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Provisioning timestamp |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Last modification timestamp |

### `memberships`

Links users to tenants with a role assignment. A user may have at most one membership per tenant.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique membership identifier |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL | Owning tenant |
| `user_id` | `UUID` | FK → `users.id`, NOT NULL | The member user |
| `role_id` | `UUID` | FK → `roles.id`, NOT NULL | Assigned role |
| `auth_methods` | `TEXT[]` | NOT NULL, DEFAULT `'{}'` | Permitted authentication methods |
| `extra_permissions` | `TEXT[]` | NOT NULL, DEFAULT `'{}'` | Additional permission keys beyond the role |
| `identifier` | `VARCHAR(255)` | | Tenant-scoped identifier (e.g., employee number) |
| `status` | `VARCHAR(32)` | NOT NULL, DEFAULT `'active'` | `active` or `suspended` |
| `metadata` | `JSONB` | DEFAULT `'{}'` | Membership-level metadata |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Creation timestamp |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Last modification timestamp |

**Unique constraint:** `(tenant_id, user_id)`

### `roles`

Named collections of permission keys scoped to a single tenant.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique role identifier |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL | Owning tenant |
| `name` | `VARCHAR(128)` | NOT NULL | Role display name |
| `description` | `TEXT` | | Human-readable description |
| `permissions` | `TEXT[]` | NOT NULL, DEFAULT `'{}'` | Permission keys granted by this role |
| `is_system` | `BOOL` | NOT NULL, DEFAULT `false` | Whether created and managed by the platform |
| `is_default` | `BOOL` | NOT NULL, DEFAULT `false` | Assigned automatically to new members |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Creation timestamp |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Last modification timestamp |

**Unique constraint:** `(tenant_id, name)`

### `sessions`

Authenticated user sessions with device and geo metadata.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique session identifier |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL | Tenant the session belongs to |
| `user_id` | `UUID` | FK → `users.id`, NOT NULL | Authenticated user |
| `started_at` | `TIMESTAMPTZ` | NOT NULL | Session creation timestamp |
| `ended_at` | `TIMESTAMPTZ` | | Null while the session is active |
| `ip_address` | `VARCHAR(45)` | | IPv4 or IPv6 address |
| `user_agent` | `TEXT` | | Raw HTTP User-Agent string |
| `device_type` | `VARCHAR(32)` | | `desktop`, `mobile`, or `tablet` |
| `auth_method` | `VARCHAR(64)` | | Authentication method used |
| `status` | `VARCHAR(32)` | NOT NULL, DEFAULT `'active'` | `active` or `ended` |
| `country` | `VARCHAR(2)` | | ISO 3166-1 alpha-2 country code |
| `city` | `VARCHAR(128)` | | City name from geo-IP lookup |
| `request_count` | `INT` | NOT NULL, DEFAULT `0` | Number of requests in this session |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Row creation timestamp |

### `service_principals`

Non-human API clients that authenticate via an API key.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique principal identifier |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL | Owning tenant |
| `name` | `VARCHAR(255)` | NOT NULL | Display name |
| `api_key_hash` | `TEXT` | NOT NULL | bcrypt hash of the raw API key |
| `api_key_prefix` | `VARCHAR(8)` | NOT NULL | First 8 characters (displayed in the UI) |
| `permissions` | `TEXT[]` | NOT NULL, DEFAULT `'{}'` | Explicit permission keys granted |
| `rate_limit` | `INT` | NOT NULL, DEFAULT `0` | Requests per minute; 0 = unlimited |
| `expires_at` | `TIMESTAMPTZ` | | Null = never expires |
| `last_used_at` | `TIMESTAMPTZ` | | Timestamp of most recent authenticated request |
| `status` | `VARCHAR(32)` | NOT NULL, DEFAULT `'active'` | `active` or `revoked` |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Creation timestamp |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Last modification timestamp |

## Tenant Schema

Each tenant gets a private PostgreSQL schema created at provisioning time. The schema name is the tenant's `schema_name` field (e.g., `tenant_acme`). The following tables are created inside every tenant schema.

<Aside type="note">
Module-specific tables (e.g., `repairs`, `vehicles`) are also created in the tenant schema by the module's migration files. Those tables are defined by each module, not by the platform core.
</Aside>

---

### `events_outbox`

Stores domain events waiting to be delivered to subscribers. The worker pool polls this table and marks events as processed.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique event identifier |
| `name` | `VARCHAR(255)` | NOT NULL | Dot-separated event name (e.g., `repairs.created`) |
| `module` | `VARCHAR(128)` | NOT NULL | Module that emitted the event |
| `tenant_id` | `UUID` | NOT NULL | Owning tenant |
| `aggregate_type` | `VARCHAR(128)` | NOT NULL | Domain entity type (e.g., `repair`) |
| `aggregate_id` | `UUID` | NOT NULL | UUID of the affected entity |
| `version` | `INT` | NOT NULL, DEFAULT `1` | Payload schema version |
| `payload` | `JSONB` | NOT NULL | Event-specific data |
| `metadata` | `JSONB` | DEFAULT `'{}'` | Cross-cutting metadata (request ID, actor) |
| `occurred_at` | `TIMESTAMPTZ` | NOT NULL | When the domain event occurred |
| `sequence` | `BIGSERIAL` | NOT NULL | Monotonically increasing; used for ordered delivery |
| `processed_at` | `TIMESTAMPTZ` | | Null = not yet delivered |
| `retry_count` | `INT` | NOT NULL, DEFAULT `0` | Number of delivery attempts |
| `last_error` | `TEXT` | | Error message from the most recent failed attempt |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Row insertion timestamp |

**Index:** `(processed_at, sequence)` — for efficient outbox polling.

---

### `audit_log`

Immutable record of every write operation performed within the tenant. Rows are never updated or deleted.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique audit entry identifier |
| `tenant_id` | `UUID` | NOT NULL | Owning tenant |
| `actor_id` | `UUID` | NOT NULL | User or service principal that performed the action |
| `action` | `VARCHAR(64)` | NOT NULL | Verb (e.g., `create`, `update`, `delete`, `approve`) |
| `resource_type` | `VARCHAR(128)` | NOT NULL | Entity type (e.g., `repair`, `user`) |
| `resource_id` | `UUID` | NOT NULL | UUID of the affected entity |
| `changes` | `JSONB` | | Before/after diff with masked fields redacted |
| `metadata` | `JSONB` | DEFAULT `'{}'` | IP address, user-agent, and other contextual data |
| `session_id` | `UUID` | | FK to the session (if user-initiated) |
| `request_id` | `VARCHAR(128)` | | HTTP request trace ID |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | When the action was recorded |

**Indexes:** `(resource_type, resource_id)`, `(actor_id)`, `(created_at DESC)`

---

### `notification_rules`

Tenant-managed rules that map domain events to delivery channels.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique rule identifier |
| `tenant_id` | `UUID` | NOT NULL | Owning tenant |
| `module` | `VARCHAR(128)` | NOT NULL | Module that owns this rule |
| `event_name` | `VARCHAR(255)` | NOT NULL | Event that triggers delivery |
| `channel` | `VARCHAR(32)` | NOT NULL | `in_app`, `email`, `slack`, or `webhook` |
| `template` | `TEXT` | NOT NULL | Template name or inline template string |
| `config` | `JSONB` | DEFAULT `'{}'` | Channel-specific config (recipient, URL, etc.) |
| `enabled` | `BOOL` | NOT NULL, DEFAULT `true` | Whether the rule is active |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Creation timestamp |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Last modification timestamp |

---

### `notifications`

In-app notifications delivered to individual users within a tenant.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique notification identifier |
| `tenant_id` | `UUID` | NOT NULL | Owning tenant |
| `user_id` | `UUID` | NOT NULL | Recipient user |
| `title` | `VARCHAR(255)` | NOT NULL | Short notification title |
| `body` | `TEXT` | NOT NULL | Full notification body |
| `link` | `TEXT` | | Optional deep-link URL |
| `read` | `BOOL` | NOT NULL, DEFAULT `false` | Whether the user has read it |
| `read_at` | `TIMESTAMPTZ` | | Timestamp the notification was read |
| `metadata` | `JSONB` | DEFAULT `'{}'` | Additional data for rendering or routing |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Delivery timestamp |

**Index:** `(user_id, read, created_at DESC)` — for efficient unread count and feed queries.

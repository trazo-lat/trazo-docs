Trazo uses a **schema-per-tenant** PostgreSQL isolation model with a **three-tier** table layout:

- The **`public` schema** holds cross-tenant tables: the tenant registry, users, memberships, roles, permission definitions, module enablement, and platform defaults for notifications and translations. These are managed by `migrations/core/public/`.
- Each **tenant schema** (`tenant_<slug>`) holds two kinds of tables:
  - **Core per-tenant subsystems** from `migrations/core/tenant/`: sessions, audit logs, events outbox, hooks, notifications, storage, search, approvals, etc.
  - **Module-owned tables** from `migrations/modules/<name>.sql`: each enabled module's domain tables (e.g., `agents`, `conversations`, `documents_*`, `entities`, `repairs`, …).

Because the tenant boundary is the schema itself, **per-tenant tables do not carry a `tenant_id` column**. Public-schema tables that hold rows for many tenants (e.g., `tenant_memberships`, `service_principals`) do keep `tenant_id` as a foreign key into `public.tenants`.

All primary keys are `UUID`. All timestamps are `TIMESTAMPTZ` stored in UTC.

---

## Public Schema (`migrations/core/public/`)

Cross-tenant tables shared by every tenant. Applied once via `POST /api/v1/admin/migrations/bootstrap`.

### `tenants`

One row per customer organization. Controls the schema namespace and branding.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique tenant identifier |
| `public_id` | `TEXT` | UNIQUE | Public-facing identifier (e.g., `ten_…`) |
| `name` | `TEXT` | NOT NULL | Display name of the organization |
| `subdomain` | `TEXT` | UNIQUE, NOT NULL | URL-safe subdomain (e.g., `acme`) |
| `schema_name` | `TEXT` | UNIQUE, NOT NULL | PostgreSQL schema name (e.g., `tenant_acme`) |
| `logo_url` | `TEXT` | NOT NULL DEFAULT `''` | URL to the tenant's logo |
| `theme` | `JSONB` | DEFAULT `'{}'` | Custom branding tokens (colors, fonts) |
| `config` | `JSONB` | DEFAULT `'{}'` | Tenant-level feature configuration |
| `status` | `TEXT` | NOT NULL, DEFAULT `'active'` | `active`, `suspended`, or `deleted` |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Provisioning timestamp |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Last modification timestamp |

### `users`

Platform-level user accounts. Users are created once and can hold memberships in multiple tenants.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique user identifier |
| `public_id` | `TEXT` | UNIQUE | Public-facing identifier (e.g., `usr_…`) |
| `email` | `TEXT` | UNIQUE | Email address; nullable for OAuth-only accounts |
| `phone` | `TEXT` | UNIQUE | E.164 phone number |
| `name` | `TEXT` | NOT NULL | Display name |
| `avatar_url` | `TEXT` | NOT NULL DEFAULT `''` | URL to the user's profile image |
| `locale` | `TEXT` | NOT NULL DEFAULT `'es'` | BCP 47 locale code |
| `timezone` | `TEXT` | NOT NULL DEFAULT `'America/Bogota'` | IANA timezone |
| `password_hash` | `TEXT` | NOT NULL DEFAULT `''` | bcrypt hash; empty for OAuth-only accounts |
| `status` | `TEXT` | NOT NULL, DEFAULT `'active'` | `active`, `invited`, or `suspended` |
| `email_verified` | `BOOLEAN` | NOT NULL, DEFAULT `false` | Whether the email has been confirmed |
| `metadata` | `JSONB` | DEFAULT `'{}'` | Arbitrary user-level metadata |
| `oauth2_issuer` | `TEXT` | nullable | Federated identity provider issuer |
| `oauth2_sub` | `TEXT` | nullable | Subject claim from the OIDC provider |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Creation timestamp |
| `updated_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Last modification timestamp |

### `tenant_memberships`

Links users to tenants with a role assignment. A user has at most one membership per tenant.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique membership identifier |
| `public_id` | `TEXT` | | Public-facing identifier |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL | Owning tenant |
| `user_id` | `UUID` | FK → `users.id`, NOT NULL | The member user |
| `role_id` | `UUID` | FK → `roles.id`, NOT NULL | Assigned role |
| `auth_methods` | `TEXT[]` | NOT NULL, DEFAULT `'{}'` | Permitted authentication methods |
| `identifier` | `TEXT` | NOT NULL DEFAULT `''` | Tenant-scoped identifier (e.g., employee number) |
| `extra_permissions` | `TEXT[]` | NOT NULL, DEFAULT `'{}'` | Additional permission keys beyond the role |
| `status` | `TEXT` | NOT NULL, DEFAULT `'active'` | `active` or `suspended` |
| `metadata` | `JSONB` | DEFAULT `'{}'` | Membership-level metadata |
| `created_at` / `updated_at` | `TIMESTAMPTZ` | NOT NULL | Lifecycle timestamps |

**Unique constraint:** `(tenant_id, user_id)`

### `roles`

Named collections of permission keys scoped to a single tenant.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique role identifier |
| `public_id` | `TEXT` | | Public-facing identifier |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL | Owning tenant |
| `name` | `TEXT` | NOT NULL | Role display name |
| `description` | `TEXT` | NOT NULL DEFAULT `''` | Human-readable description |
| `permissions` | `TEXT[]` | NOT NULL, DEFAULT `'{}'` | Permission keys granted by this role |
| `is_system` | `BOOLEAN` | NOT NULL, DEFAULT `false` | Whether created and managed by the platform |
| `is_default` | `BOOLEAN` | NOT NULL, DEFAULT `false` | Assigned automatically to new members |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Creation timestamp |

**Unique constraint:** `(tenant_id, name)`

### `tenant_invitations`

Pending invitations for users to join a tenant. Defined in `migrations/core/public/invitations.sql`.

### `tenant_modules`

One row per `(tenant, module)` pair recording which modules are enabled for which tenant. Defined in `migrations/core/public/modules.sql`. Drives the `SchemaProvisioner` when a module is enabled or disabled.

### `permission_definitions` / `scope_definitions`

Module-registered permission keys and scope filters. Synced from each module's `module.yaml` at startup.

### `service_principals`

Non-human API clients that authenticate via an API key.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique principal identifier |
| `public_id` | `TEXT` | | Public-facing identifier |
| `tenant_id` | `UUID` | FK → `tenants.id`, NOT NULL | Owning tenant |
| `name` | `TEXT` | NOT NULL | Display name |
| `api_key_hash` | `TEXT` | NOT NULL | bcrypt hash of the raw API key |
| `api_key_prefix` | `TEXT` | NOT NULL | Visible key prefix (shown in the UI) |
| `permissions` | `TEXT[]` | NOT NULL, DEFAULT `'{}'` | Explicit permission keys granted |
| `rate_limit` | `INT` | NOT NULL, DEFAULT `0` | Requests per minute; 0 = unlimited |
| `expires_at` | `TIMESTAMPTZ` | nullable | Null = never expires |
| `last_used_at` | `TIMESTAMPTZ` | nullable | Timestamp of most recent authenticated request |
| `status` | `TEXT` | NOT NULL, DEFAULT `'active'` | `active` or `revoked` |
| `created_by` | `UUID` | FK → `users.id` | Who provisioned the principal |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | Creation timestamp |

---

### `notification_rules` (platform defaults)

Platform-default rules that map events to channels. Tenants override them by inserting rows into `tenant_notification_rules` inside their own schema. Defined in `migrations/core/public/notification_rules.sql`.

### `translations` (platform/module defaults)

Platform and module default translations (scope = `platform` or `module`). Tenants override them via `tenant_translations` in their schema. Defined in `migrations/core/public/translations.sql`.

### `migration_history`

Records every applied migration by tier, target schema, file path, and content hash. Used by the diff engine to skip work that is already in sync.

### Auth-helper tables

`magic_link_tokens`, `webauthn_credentials`, `password_reset_tokens`, `discovery_codes` — short-lived auth artifacts.

## Per-Tenant Core (`migrations/core/tenant/`)

Applied to every `tenant_<slug>` schema at tenant creation. These tables hold per-tenant subsystem data managed by the platform itself, not by any single domain module.

<Aside type="note">
None of these tables carry a `tenant_id` column. The schema (`tenant_<slug>`) **is** the tenancy boundary. `search_path` routes unqualified queries to the active tenant.
</Aside>

---

### `sessions`

Authenticated user sessions with device and geo metadata. The auth middleware loads tenant context before looking up the session because this table now lives per-tenant.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique session identifier |
| `user_id` | `UUID` | FK → `public.users.id` | Authenticated user |
| `started_at` | `TIMESTAMPTZ` | NOT NULL | Session creation timestamp |
| `ended_at` | `TIMESTAMPTZ` | nullable | Null while the session is active |
| `last_active_at` | `TIMESTAMPTZ` | NOT NULL | Last request timestamp |
| `ip_address` | `TEXT` | NOT NULL DEFAULT `''` | IPv4 or IPv6 address |
| `user_agent` | `TEXT` | NOT NULL DEFAULT `''` | Raw HTTP User-Agent string |
| `device_type` | `TEXT` | NOT NULL DEFAULT `''` | `desktop`, `mobile`, or `tablet` |
| `auth_method` | `TEXT` | NOT NULL DEFAULT `'password'` | Authentication method used |
| `token_id` | `UUID` | nullable | Refresh-token family identifier |
| `status` | `TEXT` | NOT NULL, DEFAULT `'active'` | `active`, `ended`, or `revoked` |
| `revoked_reason` | `TEXT` | NOT NULL DEFAULT `''` | Reason if `status = 'revoked'` |
| `country`, `city` | `TEXT` | NOT NULL DEFAULT `''` | Geo-IP lookup result |
| `request_count` | `INT` | NOT NULL, DEFAULT `0` | Number of requests in this session |

### `audit_logs`

Immutable record of every write operation performed within the tenant. Rows are never updated or deleted.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique audit entry identifier |
| `user_id` | `UUID` | nullable | User who performed the action |
| `session_id` | `UUID` | nullable | Originating session (if user-initiated) |
| `request_id` | `TEXT` | NOT NULL DEFAULT `''` | HTTP request trace ID |
| `action` | `TEXT` | NOT NULL | Verb (e.g., `create`, `update`, `delete`, `approve`) |
| `resource_type` | `TEXT` | NOT NULL | Entity type (e.g., `repair`, `user`) |
| `resource_id` | `TEXT` | NOT NULL | Identifier of the affected entity |
| `changes` | `JSONB` | NOT NULL DEFAULT `'[]'` | RFC 6902 JSON Patch diff |
| `before_data`, `after_data` | `JSONB` | nullable | Snapshots, with masked fields redacted |
| `metadata` | `JSONB` | NOT NULL DEFAULT `'{}'` | IP, user-agent, and other contextual data |
| `created_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | When the action was recorded |

**Indexes:** `(resource_type, resource_id)`, `(user_id)`, `(created_at)`, `(action)`

### `events_outbox`

Stores domain events waiting to be delivered to subscribers. The worker pool polls this table and marks events as processed.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | NOT NULL, UNIQUE | Event identifier |
| `name` | `TEXT` | NOT NULL | Dot-separated event name (e.g., `repairs.created`) |
| `version` | `INT` | NOT NULL, DEFAULT `1` | Payload schema version |
| `module` | `TEXT` | NOT NULL | Module that emitted the event |
| `aggregate_type` | `TEXT` | NOT NULL | Domain entity type |
| `aggregate_id` | `TEXT` | NOT NULL | Affected entity identifier |
| `payload` | `JSONB` | NOT NULL DEFAULT `'{}'` | Event-specific data |
| `metadata` | `JSONB` | NOT NULL DEFAULT `'{}'` | Cross-cutting metadata |
| `occurred_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | When the event occurred |
| `sequence` | `BIGSERIAL` | PK | Monotonic order |
| `processed` | `BOOLEAN` | NOT NULL, DEFAULT `false` | Delivery state |
| `processed_at` | `TIMESTAMPTZ` | nullable | When the event was delivered |
| `error` | `TEXT` | NOT NULL DEFAULT `''` | Last delivery error |
| `retry_count` | `INT` | NOT NULL, DEFAULT `0` | Delivery attempts |
| `next_retry_at` | `TIMESTAMPTZ` | NOT NULL, DEFAULT `NOW()` | When the next retry is scheduled |
| `instance_name` | `TEXT` | NOT NULL DEFAULT `''` | Multi-instance module discriminator |

---

### `event_subscriptions`

Per-tenant subscriber registrations. Replaces in-memory subscription lists so delivery is an indexed DB lookup. Indexed by `(event_pattern)` with `enabled = TRUE`.

### `event_deliveries`

One row per (event, subscription) pair with independent retry state. Enables per-handler backoff and dead-lettering separately from the source event.

---

### `tenant_notification_rules`

Tenant-managed overrides on top of `public.notification_rules`. `MatchingRules(ctx, eventName)` returns the union; tenant overrides win on the most-specific scope.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique rule identifier |
| `public_id` | `TEXT` | | Public-facing identifier |
| `module` | `TEXT` | NOT NULL | Module that owns this rule |
| `event_name` | `TEXT` | NOT NULL | Event that triggers delivery |
| `channel` | `TEXT` | NOT NULL, DEFAULT `'in_app'` | `in_app`, `email`, `slack`, or `webhook` |
| `template` | `TEXT` | NOT NULL DEFAULT `''` | Template name or inline template |
| `config` | `JSONB` | NOT NULL DEFAULT `'{}'` | Channel-specific config |
| `enabled` | `BOOLEAN` | NOT NULL, DEFAULT `true` | Whether the rule is active |
| `created_at` / `updated_at` | `TIMESTAMPTZ` | NOT NULL | Lifecycle timestamps |

### `in_app_notifications`

In-app notifications delivered to individual users within a tenant.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | `UUID` | PK | Unique notification identifier |
| `public_id` | `TEXT` | | Public-facing identifier |
| `user_id` | `UUID` | FK → `public.users.id` | Recipient user |
| `title` | `TEXT` | NOT NULL | Short notification title |
| `body` | `TEXT` | NOT NULL DEFAULT `''` | Full notification body |
| `link` | `TEXT` | NOT NULL DEFAULT `''` | Optional deep-link URL |

## Per-Tenant Module Tables (`migrations/modules/<name>.sql`)

Each domain module ships exactly one SQL file describing all its tables. The file is applied to a tenant's schema when the module is provisioned (either at tenant creation, if listed in `tenant.default_modules`, or via the `SchemaProvisioner` when the module is enabled later).

The bundled module files create tables such as:

| Module | Tables |
|---|---|
| `agents` | `agent_configs`, `agent_usage` |
| `billing` | `billing_contracts`, `invoices`, `invoice_lines`, `invoice_series`, `payments`, `credit_notes`, `credit_note_lines` |
| `conversations` | `conversations`, `messages`, `participants`, `channel_configs` |
| `documents` | `document_types`, `document_instances`, `document_signatures`, `document_share_tokens` |
| `entities` | `entity_types`, `entities`, `entity_relation_definitions`, `entity_relations` |

<Aside type="note">
Module tables created by downstream packages (e.g., a custom `repairs` module) live in the same per-tenant schema and follow the same convention: no `tenant_id` column, bare table names in SQL.
</Aside>

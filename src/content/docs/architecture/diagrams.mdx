## HTTP Request Lifecycle

The complete middleware chain and handler flow.

```
                         HTTP Request
                             │
               ┌─────────────▼──────────────┐
               │ 1. RecoveryMiddleware       │  Catches panics → 500
               ├─────────────┬──────────────┤
               │ 2. CORSMiddleware           │  Access-Control-* headers
               ├─────────────┬──────────────┤
               │ 3. RequestIDMiddleware      │  X-Request-ID → context
               ├─────────────┬──────────────┤
               │ 4. CanonicalLogMiddleware   │  One JSON line per request
               ├─────────────┬──────────────┤
               │     Public path? ──yes──→   │  Skip auth (login, healthz...)
               │         │no                 │
               │ 5. JWTAuthMiddleware        │  Validate token, load user/
               │                             │  tenant/permissions/session
               ├─────────────┬──────────────┤
               │ 6. RateLimitMiddleware      │  Token bucket: per-IP,
               │                             │  per-tenant, path overrides
               └─────────────┬──────────────┘
                             │
               ┌─────────────▼──────────────┐
               │     Route Handler (Huma)    │
               │                             │
               │  1. Validate input          │
               │  2. MustCan(permission)     │
               │  3. Hooks.Execute(before)   │
               │  4. DB.Tx(ctx, func(tx) {   │
               │       repo.Create(tx)       │
               │       Events.EmitTx(tx)     │  ◄── same transaction
               │     })                      │
               │  5. Audit.Create(resource)  │
               │  6. Return JSON response    │
               └────────────────────────────┘
```

**Public paths** that skip auth: `/api/v1/auth/login`, `/api/v1/auth/refresh`, `/api/v1/auth/oauth2/*`, `/api/v1/auth/magic-link/*`, `/api/v1/auth/webauthn/*`, `/api/v1/invites`, `/healthz`, `/openapi`, `/docs`

## Event → Hook → Notification Pipeline

```
 MODULE SERVICE (e.g., repairs.Create)
   │
   │  core.Events.EmitTx(ctx, tx, event)   ◄── same DB transaction
   │
   ▼
 events_outbox TABLE (committed atomically with business data)
   │
   ▼
 BACKGROUND WORKER (polls every 1s, batch of 50, FOR UPDATE SKIP LOCKED)
   │
   ├── SYNC subscribers (high priority, in delivery loop)
   │     └── Domain module handlers
   │
   └── ASYNC subscribers (worker pool, size=10)
         └── Notification Service (wildcard "*", priority: -10)
               │
               ├── 1. Match notification_rules (tenant + event name)
               ├── 2. Resolve targets (by user field or role)
               ├── 3. Render template (Markdown → channel-specific)
               └── 4. Dispatch to channel:
                     │
                     ├── In-App  → stored in DB (read/unread)
                     ├── Email   → SMTP + TLS, retry ×3
                     ├── Slack   → Block Kit via webhook
                     └── Webhook → HTTPS POST, retry ×3

 RETRY: retry_count², max 5 retries → dead-letter queue
 HOOKS: tenant-scoped webhooks + expression language (expr-lang)
        with per-URL circuit breaker (threshold → cooldown → half-open)
```

## Multi-Tenancy & Schema Isolation

```
 TENANT CREATION:
   1. INSERT INTO public.tenants → schema_name = "tenant_acme"
   2. CREATE SCHEMA tenant_acme
   3. Run tenant migrations (roles, memberships tables)
   4. Seed default roles (Admin, Member, Viewer)
   5. Provision default modules → INSERT INTO tenant_modules
   6. Sync permissions from module.yaml

 REQUEST ISOLATION:
   JWTAuthMiddleware loads tenant.SchemaName from token claims
   │
   ▼
   core.DB.Tx(ctx, func(tx) {
     SET LOCAL search_path TO tenant_acme, public
     //                        ▲              ▲
     //                   tenant tables   shared tables
     SELECT * FROM repairs  →  tenant_acme.repairs
     SELECT * FROM tenants  →  public.tenants (fallback)
   })

 MULTI-INSTANCE MODULES:
   tenant_modules:
   ┌─────────────┬──────────────┬──────────────┐
   │ module_name │ instance_name│ table_prefix │
   ├─────────────┼──────────────┼──────────────┤
   │ invoices    │ invoices-in  │ invoices_in_ │
   │ invoices    │ invoices-out │ invoices_out_│
   └─────────────┴──────────────┴──────────────┘

   tenant_acme schema:
     invoices_in_invoices, invoices_in_invoice_items
     invoices_out_invoices, invoices_out_invoice_items

   Routes: GET /api/v1/tenants/{tid}/i/{instance}/invoices
   Perms:  invoices-in.read.all (derived from invoices.read.all)
   Events: invoices-in.created  (derived from invoices.created)
```

## Configuration Hierarchy

```
 STATIC CONFIG (highest wins):
   1. CLI Flags        --port 9090
   2. Environment      TRAZO_SERVER_PORT=9090
   3. trazo.yaml       server: { port: 8080 }
   4. Defaults         (hardcoded)

 RUNTIME SETTINGS (3-tier, TTL cache 60s):
   core.Config.Setting(ctx, "repairs", "auto_assign")
     1. tenant_settings    (per-tenant override)
     2. global_settings    (platform-wide)
     3. module.yaml        (module defaults)
```

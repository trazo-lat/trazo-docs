---
title: Hooks
description: Tenant-configurable webhooks and expression-based hooks for extending module behavior without code changes.
---

Hooks allow tenants to inject custom logic at defined points in a module's lifecycle without modifying the module code. Two hook types are supported: **webhooks** (external HTTP calls) and **expressions** (in-process evaluation).

Hook configuration is stored in the per-tenant `tenant_hooks` table (`tenant_<slug>.tenant_hooks`, defined by `migrations/core/tenant/hooks.sql`). Because the table lives inside the tenant schema, it has no `tenant_id` column — the tenancy is encoded in the schema name.

## Hook Types

| Type | Execution | Use Case |
|---|---|---|
| Webhook | HTTP POST to external URL | Integrations with external systems, complex logic in tenant's own backend |
| Expression | In-process via [expr-lang](https://github.com/expr-lang/expr) | Simple field transformations, conditional logic, no external dependency |

## How Hooks Work

Services call `core.Hooks.Execute()` at defined lifecycle points (e.g., `repairs.before_create`). The executor fetches the tenant's hook configuration from the database, runs it, and returns any modifications.

```go
// In service code
hookResult, _ := s.core.Hooks.Execute(ctx, "repairs.before_create", input)
if hookResult.Executed && hookResult.Modified != nil {
    for k, v := range hookResult.Modified {
        input[k] = v  // Merge hook-returned fields
    }
}
```

Hooks are **best-effort** — errors are logged but never block the business operation.

## Webhook Hooks

When a webhook hook is triggered, the platform sends a POST request:

```json
{
  "hook": "repairs.before_create",
  "tenant": { "id": "t123", "subdomain": "acme" },
  "user_id": "u456",
  "request_id": "req-789",
  "timestamp": "2026-04-02T14:30:00Z",
  "data": { /* event payload */ }
}
```

The external service responds with modifications:

```json
{
  "modified": {
    "priority": "urgent",
    "assigned_to": "user-specialist-001"
  }
}
```

### Circuit Breaker

Each webhook URL has an independent circuit breaker to prevent cascading failures:

| State | Behavior |
|---|---|
| **Closed** (normal) | Requests flow normally |
| **Open** (after N failures) | All requests immediately fail without calling the URL |
| **Half-Open** (after cooldown) | One test request allowed; success → Closed, failure → Open |

Default configuration:
- Threshold: 5 consecutive failures
- Cooldown: 60 seconds
- Retry: 3 attempts with exponential backoff (500ms initial, 5s max)
- Transient errors: 429, 5xx trigger retry
- Timeout: 10 seconds per request

### Security

- Only HTTPS URLs allowed
- SSRF protection: validates against private/reserved IP ranges
- Max payload size: 1 MB (configurable)
- Custom headers supported per webhook

## Expression Hooks

Expression hooks evaluate in-process using the [expr-lang](https://github.com/expr-lang/expr) library. No external call is needed.

```
amount > 1000 && status == "pending" ? {"priority": "high"} : {}
```

- Access event payload fields via dot notation
- Return a map of field modifications, or an empty map for no changes
- Boolean `true` means "condition matched, no modifications"
- Boolean `false` means "condition not met, skip"

## Database Schema

The `tenant_hooks` table lives **inside each tenant's schema**, so it has no `tenant_id` column. The schema name (`tenant_<slug>`) is the tenancy boundary.

```sql
CREATE TABLE tenant_hooks (
    id          UUID PRIMARY KEY,
    public_id   TEXT NOT NULL DEFAULT '',
    hook_name   TEXT NOT NULL,                 -- "repairs.before_create"
    hook_type   TEXT NOT NULL DEFAULT 'webhook', -- "webhook" or "expression"
    url         TEXT NOT NULL DEFAULT '',      -- webhook URL (empty for expressions)
    expression  TEXT NOT NULL DEFAULT '',      -- expr-lang expression
    headers     JSONB NOT NULL DEFAULT '{}',   -- custom webhook headers
    enabled     BOOLEAN NOT NULL DEFAULT true,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (hook_name)
);
```

## Configuration

```yaml
# heyllave.yaml
hooks:
  webhook_timeout: 10s
  breaker_threshold: 5
  breaker_cooldown: 60s
  max_payload_size: 1048576  # 1 MB
```

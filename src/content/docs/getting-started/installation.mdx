This guide walks through cloning both repositories, compiling the binaries, and setting up your initial configuration file.

## Clone the Repositories

Both repos should live side by side in the same parent directory. This makes it easier to use relative paths and keeps a single `heyllave.yaml` config accessible from both.

```bash
mkdir -p ~/heyllave && cd ~/heyllave

git clone https://github.com/heyllave/core-service.git
git clone https://github.com/heyllave/cli.git
```

Your directory structure should look like:

```
~/heyllave/
├── heyllave-core-service/
└── heyllave-cli/
```

---

## Build `heyllave-core-service`

<Steps>
1. Enter the repo and download dependencies:

   ```bash
   cd ~/heyllave/heyllave-core-service
   go mod download
   ```

2. Build the server binary:

   ```bash
   go build -o heyllave-server ./cmd/server
   ```

3. Verify the build:

   ```bash
   ./heyllave-server --help
   ```

   You should see the server's help output listing available flags such as `--config`, `--port`, and `--log-level`.
</Steps>

---

## Build `heyllave-cli`

<Steps>
1. Enter the repo and download dependencies:

   ```bash
   cd ~/heyllave/heyllave-cli
   go mod download
   ```

2. Build the CLI binary:

   ```bash
   go build -o heyllave ./cmd/heyllave
   ```

3. Verify the build:

   ```bash
   ./heyllave --version
   ```

   Expected output:

   ```
   heyllave version 0.x.x
   ```
</Steps>

<Aside type="tip">
  Add both binaries to your `PATH` so you can invoke them from any directory:

  ```bash
  # Add to ~/.zshrc or ~/.bashrc
  export PATH="$PATH:$HOME/heyllave/heyllave-core-service:$HOME/heyllave/heyllave-cli"
  ```

  After reloading your shell, `heyllave` and `heyllave-server` will be available globally.
</Aside>

## Configure `heyllave.yaml`

Both binaries read configuration from `heyllave.yaml`. A fully annotated example is provided in `heyllave-core-service`.

```bash
cp ~/heyllave/heyllave-core-service/heyllave.example.yaml ~/heyllave/heyllave.yaml
```

<Aside type="caution">
  The `heyllave.yaml` file contains secrets such as `auth.jwt_secret` and database credentials. Never commit it to version control. It is already listed in `.gitignore` in both repos.
</Aside>

By default both binaries look for `heyllave.yaml` in the current working directory. You can point them at a different path using the `--config` flag or the `HEYLLAVE_CONFIG` environment variable:

```bash
./heyllave-server --config /etc/heyllave/heyllave.yaml
./heyllave --config /etc/heyllave/heyllave.yaml
```

See the [Configuration](/getting-started/configuration/) guide for a complete reference of every field.

## Quick Verification

Run both binaries to confirm they are working and can read your config:

<Tabs>
  <TabItem label="CLI">
    ```bash
    cd ~/heyllave
    ./heyllave-cli/heyllave --version
    ./heyllave-cli/heyllave --help
    ```
  </TabItem>
  <TabItem label="Server">
    ```bash
    cd ~/heyllave
    ./heyllave-core-service/heyllave-server --help
    ```
  </TabItem>
</Tabs>

If either binary exits with a non-zero code, check that:

- Your `heyllave.yaml` exists at the expected path.
- All required fields (`database.url`, `auth.jwt_secret`) are set.
- The Go version is 1.24 or newer (`go version`).

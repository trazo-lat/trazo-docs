Trazo is a multi-tenant platform. Each tenant represents an independent organization with its own data, users, and configuration. When a tenant is created, Trazo automatically provisions a dedicated PostgreSQL schema (e.g., `tenant_acme`) ensuring complete data isolation between organizations.

All tenant management endpoints require an authenticated session with the `tenants:write` permission for mutating operations, or `tenants:read` for read-only access.

## Create a Tenant

Provision a new tenant. The `subdomain` field must be globally unique and is used to route requests to the correct tenant context (e.g., `acme.trazo.app`).

**`POST /api/v1/tenants`**

| Field | Type | Required | Description |
|---|---|---|---|
| `name` | string | yes | Human-readable display name of the organization |
| `subdomain` | string | yes | URL-safe slug, unique across all tenants |

**Response**

```json
{
  "id": "ten_01HQ...",
  "name": "Acme Corp",
  "subdomain": "acme",
  "schema_name": "tenant_acme",
  "status": "active",
  "created_at": "2024-03-01T12:00:00Z",
  "updated_at": "2024-03-01T12:00:00Z"
}
```

The `schema_name` field reflects the PostgreSQL schema created for this tenant. This schema is automatically migrated and contains all tenant-specific tables.

## List Tenants

Retrieve a paginated list of tenants. Use the `status` filter to narrow results to active or suspended tenants.

**`GET /api/v1/tenants`**

| Query parameter | Type | Description |
|---|---|---|
| `status` | string | Filter by status: `active`, `suspended`, or `pending` |
| `limit` | integer | Maximum number of results to return (default: `20`, max: `100`) |
| `offset` | integer | Number of records to skip for pagination (default: `0`) |

**Response**

```json
{
  "items": [
    {
      "id": "ten_01HQ...",
      "name": "Acme Corp",
      "subdomain": "acme",
      "schema_name": "tenant_acme",
      "status": "active",
      "created_at": "2024-03-01T12:00:00Z"
    }
  ],
  "total": 42,
  "limit": 20,
  "offset": 0
}
```

## Update a Tenant

Partially update a tenant's metadata. Only include fields you want to change — omitted fields are left unchanged.

**`PATCH /api/v1/tenants/{id}`**

| Field | Type | Description |
|---|---|---|
| `name` | string | Updated display name |
| `logo_url` | string | URL to the tenant's logo image |
| `theme` | object | UI theme overrides (e.g., `{"primary_color": "#FF5733"}`) |
| `config` | object | Arbitrary tenant-level configuration key-value pairs |

<Tabs>

      "theme": { "primary_color": "#FF5733" },

## Schema Provisioning

When you call `POST /api/v1/tenants`, Trazo performs the following steps synchronously before returning:

1. Inserts the tenant record into the `public.tenants` table.
2. Creates a new PostgreSQL schema named `tenant_{subdomain}`.
3. Runs all pending migrations against the new schema.

This ensures the tenant is immediately usable. If schema creation fails, the entire operation is rolled back and a `500` error is returned.

## Common Errors

| Status | Code | Description |
|---|---|---|
| `409` | `SUBDOMAIN_TAKEN` | The requested `subdomain` is already in use by another tenant. Choose a different subdomain. |
| `404` | `TENANT_NOT_FOUND` | No tenant exists with the given ID. |
| `422` | `VALIDATION_ERROR` | One or more fields failed validation (e.g., subdomain contains invalid characters). |
| `403` | `FORBIDDEN` | The authenticated user does not have `tenants:write` permission. |

**Error response shape:**

```json
{
  "code": "SUBDOMAIN_TAKEN",
  "message": "The subdomain 'acme' is already taken",
  "request_id": "req_01HQ..."
}
```

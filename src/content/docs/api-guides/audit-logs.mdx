Every time a resource is created, updated, or deleted in Trazo, an immutable audit entry is written. These entries capture who performed the action, what changed, and when — making it straightforward to meet compliance requirements and diagnose production issues.

The `changes` field in each entry contains an RFC 6902 JSON Patch array describing the exact diff between the previous and current state of the resource.

Audit log access requires the `audit:read` permission.

## List Audit Entries

Retrieve a paginated, filterable list of audit log entries. Combine filters to narrow results to a specific resource, actor, or time window.

**`GET /api/v1/audit`**

| Query parameter | Type | Description |
|---|---|---|
| `resource_type` | string | Filter by resource kind, e.g. `repair`, `user`, `tenant` |
| `resource_id` | string | Filter to entries for a specific resource instance |
| `action` | string | Filter by action: `create`, `update`, `delete` |
| `actor_id` | string | Filter to entries performed by a specific user ID |
| `from` | string | ISO 8601 timestamp — only return entries at or after this time |
| `to` | string | ISO 8601 timestamp — only return entries before or at this time |
| `limit` | integer | Maximum results to return (default: `50`, max: `200`) |
| `offset` | integer | Number of records to skip for pagination (default: `0`) |

## Understanding the Changes Field

The `changes` array follows the RFC 6902 JSON Patch specification. Each element in the array is a patch operation describing a single field change.

**Supported operations:**

| Operation | Meaning |
|---|---|
| `add` | A field was set for the first time (e.g., an optional field was populated) |
| `replace` | An existing field's value was changed |
| `remove` | A field was cleared or deleted |

**Example `changes` array:**

```json
[
  { "op": "replace", "path": "/status",       "value": "completed" },
  { "op": "replace", "path": "/completed_at", "value": "2024-03-15T14:30:00Z" },
  { "op": "add",     "path": "/resolution",   "value": "Replaced faulty component" }
]
```

The `path` uses JSON Pointer notation (RFC 6901): `/` separates path segments, and array indices are integers (e.g., `/items/0/name`).

## Reconstructing Resource History

To replay the full history of a resource and understand its state at any point in time, query all audit entries for that resource ordered chronologically, then apply the patch operations in sequence.

  ```go
  package main

  import (
      "encoding/json"
      "fmt"
  )

  // applyPatch applies a single RFC 6902 patch operation to the resource map.
  // A production implementation should use a library such as github.com/evanphx/json-patch.
  func applyPatch(resource map[string]any, op map[string]any) {
      path := op["path"].(string)
      // Strip leading slash and use as top-level key (simplified for single-depth paths)
      key := path[1:]
      switch op["op"] {
      case "add", "replace":
          resource[key] = op["value"]
      case "remove":
          delete(resource, key)
      }
  }

  func reconstructHistory(entries []AuditEntry) []map[string]any {
      resource := map[string]any{}
      snapshots := make([]map[string]any, 0, len(entries))

      for _, entry := range entries {
          for _, change := range entry.Changes {
              applyPatch(resource, change)
          }
          // Deep copy the current state as a snapshot
          snapshot := make(map[string]any, len(resource))
          for k, v := range resource {
              snapshot[k] = v
          }
          snapshot["_snapshot_at"] = entry.OccurredAt
          snapshots = append(snapshots, snapshot)
      }
      return snapshots
  }

  func main() {
      // Assume entries are fetched and sorted by occurred_at ascending
      var entries []AuditEntry
      // ... populate entries ...

      snapshots := reconstructHistory(entries)
      for _, s := range snapshots {
          data, _ := json.MarshalIndent(s, "", "  ")
          fmt.Println(string(data))
      }
  }
  ```

  ```javascript
  /**
   * Reconstruct the state of a resource at each point in its history.
   * Entries must be ordered by occurred_at ascending.
   *
   * For production use, consider a library like fast-json-patch.
   */
  function reconstructHistory(entries) {
    const resource = {};
    return entries.map(entry => {
      for (const change of entry.changes) {
        // Simplified: handles top-level paths only (e.g., /status)
        const key = change.path.slice(1); // strip leading /
        if (change.op === 'add' || change.op === 'replace') {
          resource[key] = change.value;
        } else if (change.op === 'remove') {
          delete resource[key];
        }
      }
      return { ...resource, _snapshot_at: entry.occurred_at };
    });
  }

  // Fetch all entries for a resource, then reconstruct
  const { items } = await listAuditEntries(accessToken, {
    resourceType: 'repair',
    resourceId: 'rep_01HQ...',
    limit: 200,
  });

  // Sort ascending by occurred_at
  items.sort((a, b) => a.occurred_at.localeCompare(b.occurred_at));

  const history = reconstructHistory(items);
  history.forEach(snapshot => {
    console.log(`State at ${snapshot._snapshot_at}:`, snapshot);
  });
  ```

## Common Errors

| Status | Code | Description |
|---|---|---|
| `403` | `INSUFFICIENT_PERMISSIONS` | The authenticated user does not have the `audit:read` permission. Request this permission from a tenant administrator. |
| `422` | `INVALID_DATE_RANGE` | The `from` timestamp is after the `to` timestamp. |
| `422` | `VALIDATION_ERROR` | A query parameter has an invalid format (e.g., `from` is not a valid ISO 8601 timestamp). |

**Error response shape:**

```json
{
  "code": "INSUFFICIENT_PERMISSIONS",
  "message": "You do not have permission to access audit logs",
  "request_id": "req_01HQ..."
}
```
